# ternary-noether

**Noether's theorem for discrete ternary systems.**

Derive conserved quantities from discrete symmetries acting on states defined over the ternary alphabet {-1, 0, +1}.

---

## Background

### Noether's Theorem (classical)

Emmy Noether proved in 1915 that every continuous symmetry of a physical system's Lagrangian corresponds to a conserved quantity:

| Symmetry               | Conserved quantity   |
|------------------------|----------------------|
| Time translation       | Energy               |
| Space translation      | Linear momentum      |
| Spatial rotation       | Angular momentum     |

This result underpins all of modern physics. If the equations of motion look the same regardless of when, where, or in which direction you perform an experiment, then energy, momentum, and angular momentum are each conserved — not as empirical facts but as logical necessities.

### Why discrete / ternary?

In cellular automata, lattice models, and digital physics, states live on grids where each cell holds a value from a finite alphabet. The simplest non-trivial symmetric alphabet is **{-1, 0, +1}** — the ternary alphabet. It admits negation, zero, and unity with additive symmetry, making it the minimal setting in which a Noether-style analysis is meaningful.

This crate explores what happens when you replace:

- a continuous Lagrangian → a discrete energy function over ternary states, and
- continuous symmetry groups → the finite group of ternary-preserving coordinate transformations.

Conservation laws survive the discretization: if a sequence of ternary states is generated by a rule that is invariant under translation, rotation, or reflection, then the corresponding quantity is constant across that sequence.

## API Overview

### Core Types

#### `TernaryState`

A snapshot of the system: ternary positions and momenta in {-1, 0, +1}.

```rust
use ternary_noether::TernaryState;

let state = TernaryState::new(
    vec![1, 0, -1],   // positions
    vec![0, 1, -1],   // momenta
);

println!("Energy:            {}", state.energy());
println!("Total momentum:    {}", state.total_momentum());
println!("Angular momentum:  {}", state.angular_momentum());
```

Energy is computed as `E = Σ p²/2 + Σ x²/2`, exactly analogous to a harmonic oscillator.

Values outside {-1, 0, +1} are clamped on construction.

---

#### `DiscreteSymmetry`

Finite transformations on ternary coordinate vectors.

```rust
use ternary_noether::{DiscreteSymmetry, Axis};

// Translation: shift all coords by +1 (clamped to {-1,0,+1})
let t = DiscreteSymmetry::Translation(1);
assert_eq!(t.apply(&[-1, 0, 1]), vec![0, 1, 1]);

// Reflection through the X-axis: negate x-coordinates of (x,y) pairs
let rx = DiscreteSymmetry::Reflection(Axis::X);
assert_eq!(rx.apply(&[1, -1, -1, 0]), vec![-1, -1, 1, 0]);

// Reflection through the Y-axis: negate y-coordinates of (x,y) pairs
let ry = DiscreteSymmetry::Reflection(Axis::Y);
assert_eq!(ry.apply(&[1, -1, 0, 1]), vec![1, 1, 0, -1]);

// Rotation by 90° CCW: (x,y) -> (-y,x) for each (x,y) pair
let r90 = DiscreteSymmetry::Rotation(90);
assert_eq!(r90.apply(&[1, 0]), vec![0, 1]);

// Rotation by 180°: (x,y) -> (-x,-y)
let r180 = DiscreteSymmetry::Rotation(180);
assert_eq!(r180.apply(&[1, -1]), vec![-1, 1]);
```

Coordinates are treated as (x, y) pairs for `Rotation` and `Reflection`. A trailing element in an odd-length slice is left unchanged.

---

#### `ConservedQuantity`

A named, numerical conserved quantity.

```rust
use ternary_noether::{ConservedQuantity, DiscreteSymmetry, TernaryState};

let state = TernaryState::new(vec![1, -1], vec![-1, 1]);
let sym   = DiscreteSymmetry::Translation(1);
let cq    = ConservedQuantity::derive_from_symmetry(&sym, &state);

println!("{}: {}", cq.name, cq.value);  // "momentum: 0"
```

The derivation map:

| `DiscreteSymmetry` variant  | `ConservedQuantity.name` | formula                         |
|-----------------------------|--------------------------|---------------------------------|
| `Translation(_)`            | `"momentum"`             | Σ p_i                           |
| `Rotation(_)`               | `"angular_momentum"`     | Σ (x_i · p_y_i − y_i · p_x_i) |
| `Reflection(_)`             | `"momentum"`             | Σ p_i                           |

### Symmetry-specific helpers

These mirror the canonical Noether taxonomy and provide named entry-points.

#### `TimeTranslation` → Energy

```rust
use ternary_noether::{TimeTranslation, TernaryState};

let state = TernaryState::new(vec![1, 0], vec![0, 1]);
let cq = TimeTranslation::conserved_quantity(&state);
assert_eq!(cq.name, "energy");
assert_eq!(cq.value, state.energy());
```

#### `SpaceTranslation` → Momentum

```rust
use ternary_noether::{SpaceTranslation, TernaryState};

let state = TernaryState::new(vec![1, -1, 0], vec![1, 0, -1]);
let cq = SpaceTranslation::conserved_quantity(&state);
assert_eq!(cq.name, "momentum");
// value = 1 + 0 + (-1) = 0
assert_eq!(cq.value, 0.0);
```

#### `Rotation` → Angular Momentum

```rust
use ternary_noether::{Rotation, TernaryState};

let state = TernaryState::new(vec![1, 0], vec![0, 1]);
let cq = Rotation::conserved_quantity(&state);
assert_eq!(cq.name, "angular_momentum");
// L = x*py - y*px = 1*1 - 0*0 = 1
assert_eq!(cq.value, 1.0);
```

### Verification

Check whether a sequence of states actually conserves a quantity (useful for unit-testing your own ternary dynamics):

```rust
use ternary_noether::{TernaryState, Verification};

// Build a sequence of states with constant energy
let states: Vec<TernaryState> = (0..5)
    .map(|_| TernaryState::new(vec![1, 0], vec![0, 1]))
    .collect();

assert!(Verification::verify_energy_conservation(&states));
assert!(Verification::verify_momentum_conservation(&states));
assert!(Verification::verify_angular_momentum(&states));
```

Tolerances: energy and angular momentum use ε = 0.01; momentum comparison is exact (integer arithmetic).
